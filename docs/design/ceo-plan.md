## Vision

### 10x Check

A Taiwan equity intelligence platform with a **collective intelligence data flywheel**. Every trader who uses it contributes anonymized trade outcomes back to the broker label database. Labels improve with scale — the system used by 100 traders is more accurate than one maintained by 1 person. Distribution: broker labeling API that CMoney/XQ plugin developers pay to access + sector chip heat map shared virally in LINE groups. Revenue: NT$500-2,000/month API subscriptions or per-call API fees.

The moat isn't the algorithm — it's the collective behavioral data that compounds over time and can't be replicated by incumbents without rebuilding the network.

### Platonic Ideal

A veteran Taiwan trader who has spent 10 years tracking broker branch behavior — knowing every 凱基台北 pattern, every 美林 accumulation signature — with their decade of research encoded as a living, self-updating database. Paired with an AI that matches that knowledge to current price action and generates structured decision support in seconds. Shared with a community that makes the knowledge better for everyone.

Not a CLI. Not a JSON endpoint. A **shared brain** for Taiwan active traders.

## Landscape

- **CMoney 理財寶 marketplace:** 10M+ MAU, 2.4M+ paid subscribers, 50+ strategy providers. Ready-made distribution for signal tools.
- **LINE groups:** NT$1,000-10,000/month dominant Taiwan signal subscription format. Word-of-mouth + KOL-driven distribution.
- **XQ Global:** Power-trader tool with community sharing. Skews professional. Second-priority distribution.
- **Taiwan retail traders:** 56.5% of TWSE/TPEx trading value. Massive, active audience.
- **Quant/AI startup niche:** Relatively undercrowded. CMoney/XQ adding AI features but no pure-play startup with broker-behavioral-data focus.
- **Window:** ~12-18 months before incumbents close the broker labeling feature gap.

## Accepted Scope (added to this plan)

- **Signal track record:** Store every SignalOutput JSON + snapshot price. Cron job checks actual 1/3/5-day price outcomes. Output: CSV export + statistics (win rate by confidence tier, by sector). Added to Phase 2 milestone.
- **Broker label API:** FastAPI wrapper over `broker_labels` table + signal endpoints. API key auth. Rate limiting (monthly caps). Pricing: free (10K calls/month), NT$500/month (50K calls), NT$2,000/month (500K calls). Added as Phase 3b milestone (after core CLI works).
- **Collective label curation:** Outcome submission endpoint: `POST /signals/{signal_id}/outcome`. Anonymized (no PII, no user identity). Contributes to `reversal_rate` updates via Bayesian update. Phase 4 feature (requires user base first).
- **Sector chip heat map:** Extend ScoutAgent (Phase 2) to scan TWSE sector universe. Output: text table + JSON. Sector classification from TWSE official sector codes. Note: ASCII text tables render poorly on LINE mobile — the primary sharing format should be a plain-text summary (e.g., "半導體 ↑↑↑ | 金融 ↑ | 傳產 →") rather than a full table. Success criterion adjusted to: posted in LINE group (not "renders well").

## Architecture Impact

New components required by accepted expansions:

```
signal_outcomes table (PostgreSQL):
  - signal_id UUID PRIMARY KEY
  - ticker VARCHAR(10)
  - signal_date DATE
  - confidence_score INT          -- 0-100, computed by TripleConfirmationEngine
  - action VARCHAR(10)            -- LONG | CAUTION | WATCH
  - entry_price FLOAT             -- close price at signal time
  - created_at TIMESTAMP DEFAULT NOW()
  - price_1d FLOAT NULL           -- filled by cron at T+1 close
  - price_3d FLOAT NULL           -- filled at T+3 close
  - price_5d FLOAT NULL           -- filled at T+5 close
  - outcome_1d FLOAT NULL         -- (price_1d - entry_price) / entry_price
  - outcome_3d FLOAT NULL
  - outcome_5d FLOAT NULL

Price cron job (daily, runs after TWSE close ~2:30pm):
  - Data source: FinMind TaiwanStockPrice (same as main ingestion)
  - Logic: SELECT unsettled signal_outcomes WHERE created_at <= NOW() - Xd AND price_Xd IS NULL
  - Trigger: system cron (crontab or APScheduler) at 15:30 Taiwan time (30 min after TWSE close)
  - Holiday handling: skip if TWSE trading day count has not advanced (check via TWSE calendar API or FinMind's trading_date dataset)
  - Delisted/halted tickers: mark halt_flag BOOLEAN DEFAULT FALSE = TRUE; exclude halted rows from win-rate statistics
  - Cron data source: FinMind TaiwanStockPrice (same API client as main ingestion pipeline)

FastAPI service (Phase 3b):
  - GET /broker-labels/{branch_code} → BrokerLabel JSON
  - GET /broker-labels?label=隔日沖 → list[BrokerLabel]
  - GET /signals/{ticker}?date=YYYY-MM-DD → SignalOutput JSON
  - GET /track-record?days=30 → win rate stats by confidence tier
  - API key auth middleware (X-API-Key header, keys stored in PostgreSQL api_keys table)
  - Rate limiting (monthly caps, not daily): 10,000 req/month (free), 50,000/month (NT$500/mo), 500,000/month (NT$2,000/mo)
  - API key provisioning: self-serve signup via web form → email verification → key issued automatically → Stripe/台灣Pay for paid tiers. No manual provisioning. Phase 3b launch gate: payment integration must be live before public announcement.
  Note: POST /signals/{id}/outcome (collective curation) is Phase 4 only — NOT in Phase 3b

Collective curation endpoint (Phase 4 only):
  - POST /signals/{signal_id}/outcome
    Body: { "actual_entry_price": float, "actual_exit_price": float, "hold_days": int }
  - No user identity required (anonymous)
  - Sybil attack mitigation: require paid API key for outcome submission (no free submissions);
    rate-limit to 5 submissions per API key per day; weight submissions by submitter's
    historical accuracy (submitters with >60% accurate past outcomes get 2x weight;
    new submitters get 0.5x weight until 10+ verified submissions). IP-based limits are
    insufficient alone (VPN bypass) — API key requirement is the primary defense.
  - Cross-submitter outlier check: before Bayesian update, flag submissions where
    submitted_outcome deviates >2 standard deviations from the batch median for that
    signal_id. Flag for manual review; exclude from automatic Bayesian update.
    Prevents coordinated poisoning even from paid-key accounts.
  - Halt detection: if FinMind returns no price record for a ticker on the expected
    settlement date (NULL result for TaiwanStockPrice query), check TWSE halt announcement
    API (openapi.twse.com.tw/v1/exchangeReport/TWTB4U) for that ticker + date.
    If halt confirmed: set halt_flag = TRUE. If FinMind data missing for unknown reason:
    retry 3× over 48h before marking as halt.
  - Bayesian label update: after each batch of N≥20 new outcomes for a branch,
    update reversal_rate using Beta distribution conjugate update:
    reversal_rate = (alpha_prior + num_reversals) / (alpha_prior + beta_prior + total_outcomes)
    where alpha_prior = initial reversal_count, beta_prior = initial non_reversal_count from historical data

ScoutAgent (Phase 2 expansion):
  - Input: date, sector_universe = TWSE + TPEx official sector list (~30 sectors)
  - Sector classification source: TWSE Open Data API (上市公司基本資料 endpoint includes sector codes)
  - Output: SectorHeatMap(sectors: list[SectorChipScore])
  - SectorChipScore fields: sector_name, avg_concentration_top15, avg_net_buyer_count_diff,
    positive_signal_count (tickers with confidence >= 50), total_tickers_scanned
  - Shareable output: text table formatted for terminal/LINE group paste (no image generation needed)
```

## Success Criteria (expanded)

- **Signal track record (Phase 2):** After 30 days of signal logging, LONG signals with confidence ≥70 achieve >55% positive outcome_1d rate (vs. ~45% unconditional baseline for Taiwan stocks).
- **Broker label API (Phase 3b):** 3+ external developers using the API within 30 days of launch.
- **Sector heat map (Phase 2):** Sector heat map output produced daily and shared in ≥1 Taiwan trader LINE group within 30 days.
- **Collective curation (Phase 4):** 50+ weighted outcome submissions from ≥10 distinct API keys within 60 days. Measurable: monthly reversal_rate recalculation variance for 隔日沖-labeled branches decreases by ≥20% vs. historical-only baseline (no submissions) over a 3-month comparison window.

**Important note on monetization sequencing:** The broker label API (Phase 3b) is valuable WITHOUT the collective curation flywheel — it provides access to a curated broker behavioral dataset that doesn't exist anywhere publicly. The flywheel is an enhancement that makes the dataset better over time, not a prerequisite for Phase 3b revenue. This resolves the "flywheel-before-monetization" sequencing concern.

## Codex Cross-Model Corrections (Applied 2026-03-24)

Four findings from the Codex outside voice review were accepted and applied:

1. **Pillar 3 redesign** — VolumeProfile/POC cannot be computed from daily OHLCV closes (assigns all volume to close price = meaningless resistance map). Phase 1-3: replace with `close > 20-day high * 0.99` proxy (+20 pts). Phase 4+: implement real VolumeProfile from intraday tick data. See design doc for updated formula.

2. **T+1 → D+2 execution** — 分點 data is T+1. Signal generated D+1 night → earliest execution D+2 open. Pre-phase spike success criterion updated: measure 隔日沖 reversal rate at **D+2** (not D+1). The signal is only tradeable if the pattern persists to D+2.

3. **FinMind commercial licensing** — Added to TODOS.md as P1 blocker for Phase 3b. Verify FinMind TOS permits commercial data resale before building the Broker Label API.

4. **Adjusted prices** — All historical/backtest OHLCV uses `TaiwanStockPriceAdj` (dividend/split adjusted). Raw `TaiwanStockPrice` reserved for live execution price reference only.
